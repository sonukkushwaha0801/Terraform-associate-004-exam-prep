# Terraform Configuration

This domain focuses on writing, structuring, validating, and maintaining Terraform configurations.

Unlike previous domains that focused on workflow and providers, this domain teaches how infrastructure is actually defined using Terraform language constructs.

This is the largest domain in the Terraform Associate (004) exam and frequently appears throughout certification questions.

---

## Objectives

### 4a - Use and Differentiate Resource and Data Blocks

Understand the difference between creating infrastructure and retrieving information from existing infrastructure.

### 4b - Refer to Resource Attributes and Create Cross-Resource References

Learn how Terraform resources communicate and share information through attribute references.

### 4c - Use Variables and Outputs

Understand parameterization, reusable configurations, and exposing infrastructure information.

### 4d - Understand and Use Complex Types

Learn Terraform type system including:

- list
- set
- map
- object
- tuple

### 4e - Write Dynamic Configuration Using Expressions and Functions

Use Terraform expressions, conditionals, loops, and built-in functions.

### 4f - Define Resource Dependencies in Configuration

Understand implicit and explicit dependencies.

### 4g - Validate Configuration Using Custom Conditions

Implement validation rules using:

- validation blocks
- preconditions
- postconditions

### 4h - Understand Best Practices for Managing Sensitive Data

Learn secure handling of:

- Secrets
- Passwords
- API Tokens
- Credentials
- Vault Integration

---

# Learning Outcomes

After completing this domain, you should be able to:

- Create resources
- Retrieve existing infrastructure information
- Reference resource attributes
- Build reusable Terraform configurations
- Use variables and outputs effectively
- Work with complex Terraform data types
- Use functions and expressions
- Create dynamic infrastructure
- Define dependencies correctly
- Validate user input
- Manage secrets securely
