# Infrastructure as Code (IaC) with Terraform

This domain introduces the foundational concepts of Infrastructure as Code (IaC) and explains how Terraform enables automated, repeatable, version-controlled infrastructure provisioning across multiple cloud providers and services.

## Objectives

### 1a - Explain What IaC Is

Understand the concept of Infrastructure as Code, how it differs from manual infrastructure management, and why organizations adopt it.

### 1b - Describe the Advantages of IaC Patterns

Learn the business and technical benefits of Infrastructure as Code, including consistency, repeatability, automation, collaboration, and scalability.

### 1c - Explain How Terraform Manages Multi-Cloud, Hybrid Cloud, and Service-Agnostic Workflows

Understand Terraform's provider-based architecture and how it enables infrastructure management across cloud platforms, on-premises environments, and third-party services.

---

## Learning Outcomes

After completing this domain, you should be able to:

* Define Infrastructure as Code (IaC)
* Explain declarative infrastructure management
* Differentiate manual provisioning from IaC
* Describe the benefits of IaC adoption
* Understand Terraform's multi-cloud capabilities
* Explain hybrid-cloud infrastructure management
* Understand provider-based workflows
* Recognize service-agnostic infrastructure patterns
