# Terraform Fundamentals

This domain covers the core building blocks of Terraform, including providers, provider management, multi-provider configurations, and Terraform state.

These concepts form the foundation of every Terraform deployment and are heavily tested in the Terraform Associate (004) certification exam.

---

## Objectives

### 2a - Install and Version Terraform Providers

Learn how Terraform downloads, installs, and manages provider plugins, including provider source addresses and version constraints.

### 2b - Describe How Terraform Uses Providers

Understand the role of providers, provider plugins, API communication, and resource management.

### 2c - Write Terraform Configuration Using Multiple Providers

Learn how Terraform supports multi-provider configurations and manages resources across different platforms.

### 2d - Explain How Terraform Uses and Manages State

Understand Terraform state, state files, state tracking, and how Terraform determines infrastructure changes.

---

## Learning Outcomes

After completing this domain, you should be able to:

* Explain what Terraform providers are
* Configure provider requirements
* Apply provider version constraints
* Understand provider source addresses
* Configure multiple providers
* Explain provider aliases
* Understand Terraform state
* Explain state tracking
* Understand state file usage
* Explain how Terraform determines resource changes
